Раздел 1. Компоновка блоков

Элементы компоновки управляют расположением блоков на различных устройствах.

1.1 Контейнер

Весь контент должен находиться в контейнерах .container. По умолчанию контейнер растягивается на всю страницу и задает стандартные боковые отступы.

1.2 Фиксированная ширина

Контейнеры, помещенные в .layout-fixed, становятся фиксированными по ширине, как здесь. Адаптивная ширина изменяется для устройств среднего размера (планшетов) и больших экранов.

Такой же эффект имеет модификатор .container-fixed для контейнера. В этом случае .layout-fixed не нужен.

1.3 Панели

Панели .pnl формируют «подложку» для контента.

В данном примере используются две панели:

  • первая панель темного цвета используется как родительский элемент контейнера (вместо нее можно использовать фоновую фотографию);
  • вторая панель белого цвета находится внутри контейнера с фиксированной шириной, на ней написан этот текст.

Если совместить панель и контейнер в одном элементе, то отступ панели выйдет за рамки контейнера фиксированной ширины. Используя этот метод, можно выравнять текст по ширине основной колонки.

1.4 Вертикальный отступ

По умолчанию блоки находятся вплотную друг к другу (отсутствуют вертикальные поля).

Базовый класс .block добавляет стандартные вертикальные поля.

1.5 Адаптивные сетки

Сетки используются для создания колоночной компоновки и проектируются по принципу Mobile First: сначала страница макетируется под мобильный телефон (маленький экран), затем под планшет, в последнюю очередь под большой экран.

Ячейки, совмещенные с панелями, растягиваются по высоте сетки.
